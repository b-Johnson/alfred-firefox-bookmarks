Firefox Bookmarks

An Alfred workflow to search and open your Firefox bookmarks instantly.

Keywords

Keyword Description
bm [query] Search and filter your Firefox bookmarks
bmsettings Open settings.json to configure the workflow
bmupdate Force a refresh of the bookmark cache
bmhelp Open this help file

Modifier Keys

When a bookmark is selected, press a modifier key to open it in a different app:

Key Setting Default
↩ app_default Firefox
⌘ ↩ app_cmd System default browser
⌥ ↩ app_alt not set
^ ↩ app_ctrl not set
⇧ ↩ app_shift not set
fn ↩ app_fn not set

Multi-Key Modifier Combinations

You can bind any combination of modifier keys by joining them with underscores in the settings key. The key parts are mapped directly to Alfred's combined modifier syntax. 

{
  "app_shift_ctrl": "Google Chrome",
  "app_cmd_alt":    "Safari",
  "app_cmd_shift":  ["Firefox", "Google Chrome"]
}
Key Modifier Combo
app_cmd_alt⌘ ⌥ ↩
app_cmd_ctrl⌘ ^ ↩
app_cmd_shift⌘ ⇧ ↩
app_shift_ctrl⇧ ^ ↩
app_alt_ctrl⌥ ^ ↩
app_cmd_alt_ctrl⌘ ⌥ ^ ↩

Any combination of cmd, alt, ctrl, shift, and fn is supported — just join them with underscores.

Configuration

Run bmsettings to open settings.json in your default editor. The full set of options:

{
  "firefox_profile": null,
  "app_default": "Firefox",
  "app_cmd": "Browser",
  "app_alt": null,
  "app_ctrl": null,
  "app_shift": null,
  "app_fn": null
}
Key Description
firefox_profile Absolute path to your Firefox profile directory (e.g. ~/Library/Application Support/Firefox/Profiles/abc123.default-release). Set to null to auto-detect the default-release profile.
app_default App to open bookmarks in by default. Use "Browser" for the system default browser, or an app name like "Firefox", "Safari", "Google Chrome", or "Arc".
app_cmd … app_fn Apps for modifier keys. Same values as app_default.

Any app_* value can be a JSON array to open in multiple apps at once:

"app_cmd": ["Firefox", "Safari"]

How Bookmarks are Loaded

Firefox stores bookmarks in an SQLite database called places.sqlite inside your profile directory. Because Firefox locks this file while running, the workflow copies it to a temporary location before reading.

The bookmark cache is refreshed automatically every 60 minutes (configurable via the UPDATE_EVERY_MINS workflow variable in Alfred). Run bmupdate to force an immediate refresh.

Tip: After editing settings.json, the cache will be refreshed automatically on the next search.

Troubleshooting

Problem Solution
No bookmarks appear Run bmupdate and wait a few seconds, then try again.
Wrong Firefox profile used Set firefox_profile in settings.json to the full path of the correct profile.
App not found when opening Make sure the app name in settings.json matches the exact name of the application in your /Applications folder.
